Vanilla HTML

Pure Markup

Vanilla HTML is a framework built on the concept of not requiring extra markup or class attributes to attain a layout. Just write semantic HTML with some basic knowledge of Vanilla HTML and your layout is done. Seriously, now focus on the things that are actually important, like your design. Oh yeah and Vanilla HTML is 100% responsive.

Almost Magic

GRID: Start a grid by adding class="vanilla" to an element. This is the only CSS class in the API. For a simple layout all you need is <body class="vanilla">

ROWS: Elements that are direct children of class="vanilla" will become ROWS.

COLUMNS: Direct children of ROWS will be COLUMNS. Columns are weighted depending on the tag. header, section, nav, article, main, address, and footer tags produce a larger column when placed next to any other tag. Using this technique you can create an assortment of different column layouts.

Notes

Note: To achieve optimal responsive media, do not place media elements (img, video, iframe, etc.) as direct children of rows.

Tables: It's a great idea to not use tables for grids, rows or columns, so great, in fact, NEVER do it. Just nest your tables inside a column and they'll play nice.

Chrome OS

I recently discovered the wonder of the ChromeBook, and Google's first desktop OS: Chrome. Let me start by saying that unless you want to fork out several hundred pounds, you cannot get the Chrome OS. But Hexxah, offers a very good alternative, based around Linux Ubuntu. Although you don't need a Google account, when you boot into "Chrome OS", it does help, as it allows for better use and personalisation. Take note though, it must be installed onto a USB Disk, or run virtually in a VM machine, (Check out the free VirtualBox here!) And it can only be used in "Trial More", meaning that it cannot be installed and used like a regular OS, like Windows.



ChromeBook Apps


When signed you can access and download apps from Google's extensive Chrome Apps Store, however, if you don't sign in, and just use the "Guest" account, anything you do gets saved in a temporary folder, and on reboot will get deleted. So if you plan on using apps etc, and not just browsing the web, it would probably be worth signing in when it asks. Another thing worth a mention is that this "Theme" if you will, (As it is not actually Google's Chrome OS) is that it is lightning fast, if you currently own a Windows based PC/Laptop, and boot times are ticking over to a minute, you might want to give this a try, on my system it took only 28 seconds to boot, a full second quicker than a MacBook Pro, with specs about 4 times better than my own. This system is a speed demon, just wanting to be released, released upon the web. It's a shame that it isn't pratical to have the system installed full-time, becasuse otherwise I would seriously suggest installing it.


Personally, on my machine, specs being: 2.2Ghz Dual-Core Intel Celeron, 2GB RAM, it's worth keeping a copy of this lying around somewhere...
